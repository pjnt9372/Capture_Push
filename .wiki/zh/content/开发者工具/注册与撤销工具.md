## 简介
Capture_Push 采用插件化架构，所有院校适配模块均以插件形式存在。本文档说明如何通过 GUI 界面或手动方式进行插件的安装、卸载与管理。

> **注意**：原有的 `register_or_undo.py` 工具已被废弃，所有模块管理功能均由 `PluginManager` 和 GUI 界面接管。

## GUI 插件管理

系统内置了图形化的插件管理界面，位于"插件管理"标签页。

### 在线安装
1. 打开应用，进入"插件管理"标签页。
2. 系统会自动从 GitHub 获取最新的插件列表。
3. 找到目标院校，点击"下载"或"更新"按钮。
4. 系统将自动下载 ZIP 包、校验并解压到插件目录。

### 本地安装
1. 在"插件管理"标签页中，选择"从文件安装"（如果支持）或参考手动安装步骤。
2. 目前 GUI 主要支持从远程源安装，本地安装建议使用手动方式。

### 卸载插件
1. 在已安装列表中找到对应插件。
2. 点击"卸载"按钮。
3. 系统将删除对应的插件目录。

## 手动管理插件

对于开发者或网络环境受限的用户，可以通过文件操作直接管理插件。

### 插件目录位置
所有插件均存储在项目根目录下的 `core/plugins/` 文件夹中。
每个插件对应一个以院校代码命名的子文件夹（例如 `12345`）。

### 手动安装步骤
1. 获取插件 ZIP 包（通常命名为 `school_<code>_plugin.zip`）。
2. 解压 ZIP 包。
3. 将解压得到的文件夹（文件夹名应为院校代码，如 `10546`）复制到 `core/plugins/` 目录下。
4. 最终路径结构应为：`core/plugins/10546/__init__.py`。
5. 重启应用或在 GUI 中点击"刷新"即可识别新插件。

### 手动卸载步骤
1. 关闭应用程序。
2. 导航到 `core/plugins/` 目录。
3. 删除对应的院校代码文件夹（例如 `10546`）。
4. 重启应用。

## 插件结构验证

安装完成后，请确保插件目录结构正确：

```
core/plugins/
├── __init__.py          # 插件包初始化（必须存在）
├── plugin_manager.py    # 插件管理器（核心文件，勿删）
├── 12345/               # 示例院校插件
│   ├── __init__.py      # 必须包含 SCHOOL_NAME 和 PLUGIN_VERSION
│   ├── getCourseGrades.py
│   └── getCourseSchedule.py
└── [其他院校代码]/
```

## 常见问题

**Q: 安装了插件但未显示？**
A: 请检查文件夹名称是否与 `__init__.py` 中定义的院校代码一致，且路径层级正确（不要多一层文件夹）。

**Q: 如何开发新插件？**
