## 应用程序打包

项目使用 **Inno Setup** 制作 Windows 安装程序。根目录下提供了两个 `.iss` 脚本文件。

### 前置要求
- 安装 [Inno Setup 6.x](https://jrsoftware.org/isdl.php)
- 准备好 Python 环境和项目依赖（用于运行）

### 构建步骤
1. **Capture_Push_Setup.iss (完整版)**
   - 此脚本会打包应用程序文件以及可能的嵌入式环境（需自行配置源路径）。
   - 适用于需要分发完整运行环境的场景。
   - 使用 Inno Setup Compiler 打开 `.iss` 文件，检查 `Source` 路径配置，点击 "Build" -> "Compile"。

2. **Capture_Push_Lite_Setup.iss (轻量版)**
   - 仅打包应用程序源代码和资源文件。
   - 假设用户系统已安装 Python 并配置了环境。
   - 编译后生成的安装包体积较小。

### 输出产物
编译成功后，安装程序（`.exe`）将生成在 `Output` 目录中（具体位置见 `.iss` 配置）。

## 插件打包发布

院校插件是独立发布的组件，通常打包为 `.zip` 文件供用户下载或通过 PluginManager 更新。

### 打包规范
1. **目录结构**
   创建一个以**院校代码**命名的文件夹（例如 `10546`），包含以下文件：
   - `__init__.py`: 包含元数据 (`SCHOOL_NAME`, `PLUGIN_VERSION`)
   - `getCourseGrades.py`: 成绩获取逻辑
   - `getCourseSchedule.py`: 课表获取逻辑
   - 其他辅助文件（可选）

2. **压缩**
   将该文件夹压缩为 `.zip` 格式。
   - 推荐命名格式：`school_<code_plugin.zip` (例如 `school_10546_plugin.zip`)。
   - **注意**：压缩包根目录下应直接包含院校代码文件夹（例如 `zip/10546/...`），而不是散乱的文件。

### 发布流程
1. 将生成的 `.zip` 文件上传至 GitHub Releases。
2. 更新插件索引文件（如果使用中心化索引）或通知用户手动下载。

## 依赖管理

项目依赖在以下文件中声明：
- `requirements.txt`: 运行时依赖
- `pyproject.toml`: 项目元数据和构建依赖

在打包前，请确保本地环境依赖已更新：
```bash
pip install -r requirements.txt
```
