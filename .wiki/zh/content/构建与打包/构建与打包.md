## 简介
本文件概述 Capture_Push 项目的构建与打包体系。项目采用混合架构，结合 Python 核心业务逻辑和 C++ 托盘程序，通过 Inno Setup 生成最终安装包。

## 构建体系
项目构建体系主要由以下部分组成：

1.  **Python 环境准备**：依赖管理和环境配置。
2.  **C++ 托盘编译**：使用 CMake 和 MSVC 编译原生托盘程序。
3.  **Inno Setup 打包**：将所有组件整合为安装程序。

### 废弃工具说明
注意：旧版的构建脚本（如 `developer_tools/build.py` 和 `build_installer_exe.py`）已不再维护。现在的构建流程更加依赖标准工具（pip, CMake, ISCC）和 CI/CD 自动化。

## 核心组件
- **Python 核心 (`core/`, `gui/`)**: 项目的业务逻辑和界面。
- **C++ 托盘 (`tray/`)**: 提供系统托盘集成和后台常驻功能。
- **Inno Setup 脚本 (`*.iss`)**:
    - `Capture_Push_Setup.iss`: 完整版安装包，包含嵌入式 Python 环境。
    - `Capture_Push_Lite_Setup.iss`: 轻量版安装包，仅包含代码，依赖用户系统 Python。
- **配置文件**:
    - `VERSION`: 项目版本号。
    - `config.ini`: 默认配置模板。

## 构建流程

### 1. 环境准备
- 安装 Python 依赖：`pip install -r requirements.txt`
- 准备嵌入式 Python 包（仅完整版需要）。

### 2. 编译托盘程序
使用 CMake 构建 C++ 托盘程序：
```bash
cd tray
cmake -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release
```

### 3. 生成安装包
使用 Inno Setup 编译器 (ISCC) 生成安装包：
```bash
ISCC Capture_Push_Setup.iss
```
或生成轻量版：
```bash
ISCC Capture_Push_Lite_Setup.iss
```

## 结论
Capture_Push 的构建过程模块化且清晰。通过解耦各个组件的构建步骤，提高了系统的可维护性和扩展性。开发者可以根据需要单独构建托盘程序或更新 Python 代码，并通过 Inno Setup 快速生成发布版本。
