# Core目录构建流程

## 流程说明

在构建过程中，build.py 对core目录的处理遵循以下原则：

1. 复制core目录下的所有文件
2. 复制core目录下的所有子目录
3. 对plugins目录特殊处理，只保留示例插件（12345）
4. 其他学校插件通过插件管理器在线下载

## 构建流程图

```mermaid
graph TD
    A[开始构建] --> B[检查 core 目录]
    B --> C{core 目录存在?}
    C -->|否| D[结束]
    C -->|是| E[创建目标 core 目录]
    E --> F[遍历源 core 目录内容]
    F --> G{当前项是文件?}
    G -->|是| H[复制文件到目标目录]
    G -->|否| I{当前项是目录?}
    I -->|是| J{目录名是 plugins?}
    J -->|是| K[特殊处理plugins目录<br/>只复制框架和示例插件12345]
    J -->|否| L[复制整个目录到目标位置]
    H --> M[继续处理下一个项目]
    K --> M
    L --> M
    M --> N{还有更多项目?}
    N -->|是| F
    N -->|否| O[构建完成]
    D --> O
    K --> P[记录：其他学校插件将在线下载]
    P --> O
```

## 设计理念

- **完整复制**：复制core目录下的所有文件和子目录
- **插件化管理**：plugins目录只包含框架和示例插件
- **按需下载**：实际使用时通过插件管理器下载对应的学校插件
- **易于扩展**：新的学校插件可以通过插件管理器动态添加
